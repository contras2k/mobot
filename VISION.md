ICP  
ICP: владелец сайта из сферы малого бизнеса, допускающего размещение отзывов клиентов  
Контекст: нужно быстро просматривать новоразмещённые отзывы клиентов и определять их дальнейший маршрут (разместить на сайте, отправить внутренним службам, отклонить)  
Главная цель: мобильное модерирование Web 2.0 компонента сайта, оперативная реакция на жалобы клиентов  
Альтернативы: плагины CMS, e-mail, имитация отзывов  
Барьеры: нехватка времени, неудобство формата, не достучаться до ЦА  

Ценность:  
Для ICP, который сталкивается с нехваткой временного ресурса на адекватное и оперативное модерирование отзывов на сайте,  
мой ассистент пришлёт в телеграм свежеразмещённый отзыв, с указанием раздела и/или продукта, на которые был оставлен этот отзыв,  
и предложит на выбор 4 быстрых действия: Разрешить публикацию на сайте, Направить в разбор, Отклонить, Отложить.  

Сценарий 1. В очереди есть новый отзыв, который можно опубликовать  
Given В очереди есть новый "хороший" отзыв  
When /publicate  
Then помечается в базе, как разрешённый к публикации  

Сценарий 2. В очереди есть новый отзыв, который можно опубликовать  
Given В очереди есть новый "плохой" отзыв  
When /decline  
Then помечается в базе, как запрещённый к публикации  

Сценарий 3. В очереди есть новый отзыв, который требует разбирательств  
Given В очереди есть новый "проблемный" отзыв  
When /escalate  
Then помечается в базе, как требующий внимания администраторов  

Сценарий 4. Нештатный - в очереди есть новый отзыв с неясным статусом  
Given В очереди есть новый "непонятный" отзыв  
When /delay  
Then отзыв перемещается в конец очереди обработки  

Риски  
Риск: ошибочный выбор действия  
Вероятность: средняя  
Влияние: среднее  
Меры: возможность просмотреть и скорректировать 10 последних обработанных отзывов  


Метрики  
TTV: 1 секунда  
Доля сценариев, закончившаяся артефактом Then: 100%
